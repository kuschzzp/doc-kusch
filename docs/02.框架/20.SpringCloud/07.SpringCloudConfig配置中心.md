# Spring Cloud Config 配置中心

官方文档：[Spring Cloud Config](https://docs.spring.io/spring-cloud-config/reference/)

Spring Cloud Config 解决的是配置集中管理。

它通常由两部分组成：

```text
Config Server  读取配置仓库，对外提供配置
Config Client  应用启动时从 Config Server 拉配置
```

## Config Server

引入依赖：

```xml
<dependency>
  <groupId>org.springframework.cloud</groupId>
  <artifactId>spring-cloud-config-server</artifactId>
</dependency>
```

启动类：

```java
@EnableConfigServer
@SpringBootApplication
public class ConfigServerApplication {
    public static void main(String[] args) {
        SpringApplication.run(ConfigServerApplication.class, args);
    }
}
```

配置 Git 仓库：

```yaml
server:
  port: 8888

spring:
  cloud:
    config:
      server:
        git:
          uri: https://github.com/example/config-repo.git
          default-label: main
```

## 配置文件命名规则

Config Server 常见访问格式：

```text
/{application}/{profile}/{label}
```

比如：

```text
/user-service/dev/main
```

对应配置仓库里可能有：

```text
user-service-dev.yml
user-service.yml
application.yml
```

## Config Client

客户端依赖：

```xml
<dependency>
  <groupId>org.springframework.cloud</groupId>
  <artifactId>spring-cloud-starter-config</artifactId>
</dependency>
```

Spring Boot 新版本更推荐通过 `spring.config.import` 引入 Config Server：

```yaml
spring:
  application:
    name: user-service
  config:
    import: optional:configserver:http://localhost:8888
  profiles:
    active: dev
```

`optional:` 表示 Config Server 不可用时，应用不一定启动失败。生产环境是否 optional，要按实际需求决定。

## 动态刷新

如果要动态刷新配置，通常还会配合 Actuator。

```xml
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>
```

暴露刷新端点：

```yaml
management:
  endpoints:
    web:
      exposure:
        include: refresh,health,info
```

需要刷新的 Bean：

```java
@RefreshScope
@RestController
public class DemoController {
    @Value("${demo.message:default}")
    private String message;

    @GetMapping("/message")
    public String message() {
        return message;
    }
}
```

刷新：

```bash
curl -X POST http://localhost:8080/actuator/refresh
```

## 和 Nacos Config 怎么选？

| 方案 | 特点 |
| --- | --- |
| Spring Cloud Config | 官方 Spring Cloud 方案，适合 Git 管理配置 |
| Nacos Config | 注册中心和配置中心一体化，国内项目常见 |

如果团队已经有 GitOps 配置管理习惯，Spring Cloud Config 很自然。

如果团队已经大量使用 Nacos 做注册中心，用 Nacos Config 会更顺。

## 小结

Spring Cloud Config 的核心链路：

```text
Git 配置仓库 -> Config Server -> Config Client -> 应用配置
```

它适合集中管理配置，但敏感信息仍然要认真处理，不要随便明文提交到 Git。
