# OpenFeign 服务调用

官方文档：[Spring Cloud OpenFeign](https://docs.spring.io/spring-cloud-openfeign/docs/current/reference/html/)

OpenFeign 是声明式 HTTP 客户端。

以前我们调用远程服务可能会写一堆 `RestTemplate` 代码。Feign 的思路是：把远程 HTTP 调用写成 Java 接口。

## 引入依赖

```xml
<dependency>
  <groupId>org.springframework.cloud</groupId>
  <artifactId>spring-cloud-starter-openfeign</artifactId>
</dependency>
```

启动类开启 Feign：

```java
@EnableFeignClients
@SpringBootApplication
public class OrderApplication {
    public static void main(String[] args) {
        SpringApplication.run(OrderApplication.class, args);
    }
}
```

## 定义 FeignClient

```java
@FeignClient(name = "user-service")
public interface UserClient {

    @GetMapping("/users/{id}")
    UserDTO getById(@PathVariable("id") Long id);
}
```

调用：

```java
@Service
public class OrderService {
    private final UserClient userClient;

    public OrderService(UserClient userClient) {
        this.userClient = userClient;
    }

    public UserDTO getOrderUser(Long userId) {
        return userClient.getById(userId);
    }
}
```

`name = "user-service"` 对应注册中心里的服务名。

如果你配置了 Nacos、Eureka、Consul 等服务发现，Feign 会结合 Spring Cloud LoadBalancer 找到具体实例。

## url 和 name 的区别

指定服务名：

```java
@FeignClient(name = "user-service")
```

适合走注册中心和负载均衡。

指定固定地址：

```java
@FeignClient(name = "user-api", url = "http://localhost:8081")
```

适合本地调试或调用固定第三方地址。

实际项目里，内部服务优先用服务名，外部系统可以用 url。

## 超时配置

Feign 底层请求一定要配超时，不要无限等。

```yaml
spring:
  cloud:
    openfeign:
      client:
        config:
          default:
            connectTimeout: 2000
            readTimeout: 5000
          user-service:
            connectTimeout: 1000
            readTimeout: 3000
```

`default` 是全局默认。

`user-service` 是针对某个 FeignClient 的配置。

## 日志配置

Java 配置：

```java
@Configuration
public class FeignConfig {
    @Bean
    Logger.Level feignLoggerLevel() {
        return Logger.Level.BASIC;
    }
}
```

配置日志级别：

```yaml
logging:
  level:
    com.example.client.UserClient: DEBUG
```

Feign 的日志级别常见有：

```text
NONE
BASIC
HEADERS
FULL
```

生产环境别轻易开 FULL，可能把请求体、响应体、敏感信息都打出来。

## fallback

OpenFeign 可以和 Spring Cloud CircuitBreaker 集成。

开启：

```yaml
spring:
  cloud:
    openfeign:
      circuitbreaker:
        enabled: true
```

定义 fallback：

```java
@FeignClient(name = "user-service", fallback = UserClientFallback.class)
public interface UserClient {
    @GetMapping("/users/{id}")
    UserDTO getById(@PathVariable("id") Long id);
}

@Component
public class UserClientFallback implements UserClient {
    @Override
    public UserDTO getById(Long id) {
        return new UserDTO(id, "unknown");
    }
}
```

fallback 不是万能兜底。它适合返回默认值、降级提示，不适合掩盖所有错误。

## 常见坑

### 1. 忘了 @EnableFeignClients

接口写了，Bean 没生成，启动报找不到注入对象。

### 2. PathVariable 没写 value

建议明确写：

```java
@PathVariable("id") Long id
```

不要依赖参数名推断。

### 3. 超时不配

远程服务卡住时，调用方线程也被拖住。

### 4. fallback 里又调用远程服务

fallback 应该简单、稳定。不要在 fallback 里继续做复杂远程调用。

## 小结

OpenFeign 的价值是让服务调用更像本地接口。

但它本质还是 HTTP 调用，所以要关心：

```text
服务名
超时
日志
熔断
fallback
```
