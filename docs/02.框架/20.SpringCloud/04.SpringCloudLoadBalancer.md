# Spring Cloud LoadBalancer

官方文档：[Spring Cloud LoadBalancer](https://docs.spring.io/spring-cloud-commons/reference/spring-cloud-commons/loadbalancer.html)

Spring Cloud LoadBalancer 是客户端负载均衡组件。

它替代的是老项目里常见的 Ribbon。

## 客户端负载均衡是什么意思？

服务注册中心里有多个实例：

```text
user-service
  192.168.1.10:8081
  192.168.1.11:8081
  192.168.1.12:8081
```

调用方请求：

```text
http://user-service/users/1
```

LoadBalancer 会从服务发现组件里拿到实例列表，然后在客户端选一个实例发请求。

## 引入依赖

```xml
<dependency>
  <groupId>org.springframework.cloud</groupId>
  <artifactId>spring-cloud-starter-loadbalancer</artifactId>
</dependency>
```

如果使用 OpenFeign，通常会自动结合 LoadBalancer 使用。

如果自己用 `RestTemplate`、`RestClient`、`WebClient`，需要加 `@LoadBalanced`。

## RestTemplate 示例

```java
@Configuration
public class RestTemplateConfig {
    @Bean
    @LoadBalanced
    public RestTemplate restTemplate() {
        return new RestTemplate();
    }
}
```

使用：

```java
@Service
public class UserService {
    private final RestTemplate restTemplate;

    public UserService(RestTemplate restTemplate) {
        this.restTemplate = restTemplate;
    }

    public String getUser(Long id) {
        return restTemplate.getForObject("http://user-service/users/" + id, String.class);
    }
}
```

这里的 `user-service` 不是域名，而是服务名。

## WebClient 示例

```java
@Configuration
public class WebClientConfig {
    @Bean
    @LoadBalanced
    public WebClient.Builder webClientBuilder() {
        return WebClient.builder();
    }
}
```

使用：

```java
@Service
public class UserClient {
    private final WebClient.Builder builder;

    public UserClient(WebClient.Builder builder) {
        this.builder = builder;
    }

    public Mono<String> getUser(Long id) {
        return builder.build()
                .get()
                .uri("http://user-service/users/{id}", id)
                .retrieve()
                .bodyToMono(String.class);
    }
}
```

## 多个 RestTemplate 怎么办？

如果项目里既有普通 `RestTemplate`，又有负载均衡 `RestTemplate`，要区分清楚。

```java
@Configuration
public class RestTemplateConfig {
    @Bean
    @Primary
    public RestTemplate restTemplate() {
        return new RestTemplate();
    }

    @Bean
    @LoadBalanced
    public RestTemplate loadBalancedRestTemplate() {
        return new RestTemplate();
    }
}
```

注入时：

```java
@Autowired
private RestTemplate restTemplate;

@Autowired
@LoadBalanced
private RestTemplate loadBalancedRestTemplate;
```

普通 HTTP 地址用普通 `RestTemplate`。

服务名调用用 `@LoadBalanced` 的那个。

## 默认策略

Spring Cloud LoadBalancer 默认提供轮询策略，也支持随机、权重、区域优先、同实例偏好等扩展方式。

一般项目里，默认轮询已经够用。

如果要做灰度、权重、同机房优先，才需要自定义 `ServiceInstanceListSupplier` 或 LoadBalancer 配置。

## 常见坑

### 1. 用服务名调用，但没加 @LoadBalanced

```java
restTemplate.getForObject("http://user-service/users/1", String.class)
```

如果这个 `RestTemplate` 没有 `@LoadBalanced`，它会把 `user-service` 当普通域名解析，然后失败。

### 2. 服务注册中心没实例

LoadBalancer 不是注册中心。它只是从 DiscoveryClient 里拿实例。

如果 Nacos/Eureka 里没有健康实例，LoadBalancer 也选不出来。

### 3. Ribbon 老配置不生效

老项目里有很多 `ribbon.*` 配置。迁移后要检查是否已经换成 Spring Cloud LoadBalancer 的配置方式。

## 小结

Spring Cloud LoadBalancer 解决的是：

```text
服务名 -> 服务实例列表 -> 选择一个实例 -> 发请求
```

内部服务调用不要写死 IP，优先用服务名配合注册中心和 LoadBalancer。
