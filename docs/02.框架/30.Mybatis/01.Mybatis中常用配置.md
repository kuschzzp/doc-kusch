# MyBatis 中常用配置

官方文档：[MyBatis 3 - Configuration](https://mybatis.org/mybatis-3/configuration)

MyBatis 的全局配置一般写在 `mybatis-config.xml` 里。

如果是 Spring Boot 项目，很多配置会被放到 `application.yml` 里，但底层含义还是 MyBatis 这些配置项。

## 配置文件的大概结构

官方给出的顶层结构大致是这样：

```xml
<configuration>
  <properties/>
  <settings/>
  <typeAliases/>
  <typeHandlers/>
  <objectFactory/>
  <plugins/>
  <environments/>
  <databaseIdProvider/>
  <mappers/>
</configuration>
```

日常项目里最常碰到的是：

- `properties`
- `settings`
- `typeAliases`
- `typeHandlers`
- `mappers`

## properties：抽出环境变量

`properties` 用来把容易变化的配置抽出去，比如数据库连接信息。

```xml
<properties resource="jdbc.properties"/>

<environments default="development">
  <environment id="development">
    <transactionManager type="JDBC"/>
    <dataSource type="POOLED">
      <property name="driver" value="${driver}"/>
      <property name="url" value="${url}"/>
      <property name="username" value="${username}"/>
      <property name="password" value="${password}"/>
    </dataSource>
  </environment>
</environments>
```

`jdbc.properties`：

```properties
driver=com.mysql.cj.jdbc.Driver
url=jdbc:mysql://localhost:3306/test?useSSL=false&serverTimezone=Asia/Shanghai
username=root
password=123456
```

官方文档里有一个细节：如果同一个属性在多个地方都出现，优先级是：

```text
方法参数传入的 properties 最高
properties 的 resource/url 次之
properties 标签体内写的最低
```

这个优先级在多环境配置里很有用。

## settings：影响 MyBatis 运行行为

`settings` 是 MyBatis 里最值得看的一块。它不是写 SQL 的地方，而是控制 MyBatis 运行方式。

常用配置：

```xml
<settings>
  <setting name="mapUnderscoreToCamelCase" value="true"/>
  <setting name="cacheEnabled" value="true"/>
  <setting name="lazyLoadingEnabled" value="false"/>
  <setting name="autoMappingBehavior" value="PARTIAL"/>
  <setting name="defaultExecutorType" value="SIMPLE"/>
  <setting name="logImpl" value="SLF4J"/>
</settings>
```

几个常见项说明：

| 配置 | 作用 | 默认值 |
| --- | --- | --- |
| `mapUnderscoreToCamelCase` | 下划线字段自动映射驼峰属性，比如 `user_name -> userName` | `false` |
| `cacheEnabled` | 是否全局开启 mapper 中配置的缓存 | `true` |
| `lazyLoadingEnabled` | 是否开启懒加载 | `false` |
| `autoMappingBehavior` | 自动映射策略，常见为 `PARTIAL` | `PARTIAL` |
| `defaultExecutorType` | 默认执行器，`SIMPLE`、`REUSE`、`BATCH` | `SIMPLE` |
| `logImpl` | 指定日志实现，比如 `SLF4J`、`STDOUT_LOGGING` | 自动发现 |

我平时最常改的是 `mapUnderscoreToCamelCase` 和 `logImpl`。

如果项目表字段都是 `user_name`、`create_time` 这种风格，开启驼峰映射会省很多 `resultMap` 配置。

## typeAliases：少写全限定类名

`typeAliases` 只是给 Java 类型起短名字。

```xml
<typeAliases>
  <typeAlias alias="User" type="com.example.domain.User"/>
</typeAliases>
```

这样 mapper 里就可以写：

```xml
<select id="selectById" resultType="User">
  select * from user where id = #{id}
</select>
```

也可以按包扫描：

```xml
<typeAliases>
  <package name="com.example.domain"/>
</typeAliases>
```

如果类上有 `@Alias`，会优先使用注解里的名字。

## typeHandlers：Java 类型和 JDBC 类型之间的转换

MyBatis 设置参数、读取结果时，会通过 TypeHandler 做类型转换。

大多数类型不用管，MyBatis 已经内置了很多处理器，比如：

- `String`
- `Integer`
- `Long`
- `BigDecimal`
- `Date`
- `LocalDateTime`
- `Enum`

如果数据库里存的是特殊格式，比如 JSON 字符串、枚举码值，就可能需要自定义 TypeHandler。

注册方式：

```xml
<typeHandlers>
  <typeHandler handler="com.example.mybatis.JsonTypeHandler"/>
</typeHandlers>
```

## mappers：告诉 MyBatis 去哪里找 SQL

`mappers` 用来注册 mapper。

按资源文件：

```xml
<mappers>
  <mapper resource="mapper/UserMapper.xml"/>
</mappers>
```

按接口类：

```xml
<mappers>
  <mapper class="com.example.mapper.UserMapper"/>
</mappers>
```

按包扫描：

```xml
<mappers>
  <package name="com.example.mapper"/>
</mappers>
```

Spring Boot 项目里通常用 `@MapperScan`，但理解 `mappers` 仍然有必要。MyBatis 最终还是要知道 mapper 接口和 XML 在哪里。

## 一个比较常见的配置

```xml
<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE configuration
  PUBLIC "-//mybatis.org//DTD Config 3.0//EN"
  "https://mybatis.org/dtd/mybatis-3-config.dtd">

<configuration>
  <properties resource="jdbc.properties"/>

  <settings>
    <setting name="mapUnderscoreToCamelCase" value="true"/>
    <setting name="logImpl" value="SLF4J"/>
  </settings>

  <typeAliases>
    <package name="com.example.domain"/>
  </typeAliases>

  <environments default="development">
    <environment id="development">
      <transactionManager type="JDBC"/>
      <dataSource type="POOLED">
        <property name="driver" value="${driver}"/>
        <property name="url" value="${url}"/>
        <property name="username" value="${username}"/>
        <property name="password" value="${password}"/>
      </dataSource>
    </environment>
  </environments>

  <mappers>
    <package name="com.example.mapper"/>
  </mappers>
</configuration>
```

## 小结

MyBatis 配置不用死记。

先记住几块：

```text
properties  管环境变量
default settings 管运行行为
typeAliases  简化类型名
typeHandlers 管类型转换
mappers      注册 SQL 映射
```

实际项目里，大部分问题都和 `settings`、`mappers`、`resultMap` 有关。
