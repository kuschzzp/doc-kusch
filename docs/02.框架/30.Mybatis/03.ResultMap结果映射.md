# ResultMap 结果映射

官方文档：[MyBatis 3 - Mapper XML Files / resultMap](https://mybatis.org/mybatis-3/sqlmap-xml.html)

`resultMap` 是 MyBatis 里最值得认真看的功能之一。

简单查询用 `resultType` 就行，但只要遇到字段名不一致、对象嵌套、一对多关系，`resultMap` 就绕不开。

## 最简单的 resultMap

表字段：

```sql
id, user_name, email
```

Java 类：

```java
public class User {
    private Long id;
    private String userName;
    private String email;
}
```

XML：

```xml
<resultMap id="UserResultMap" type="com.example.domain.User">
  <id property="id" column="id"/>
  <result property="userName" column="user_name"/>
  <result property="email" column="email"/>
</resultMap>

<select id="selectById" resultMap="UserResultMap">
  select id, user_name, email
  from user
  where id = #{id}
</select>
```

`id` 标签用来标识主键字段，`result` 标签用来映射普通字段。

主键字段最好明确写 `id`，不要都写成 `result`。MyBatis 在处理嵌套映射和缓存时会用到这个信息。

## 开启驼峰映射后还要 resultMap 吗？

如果只是：

```text
user_name -> userName
create_time -> createTime
```

可以通过全局配置解决：

```xml
<settings>
  <setting name="mapUnderscoreToCamelCase" value="true"/>
</settings>
```

这样简单查询就不一定要写 resultMap。

但如果是下面这些情况，还是建议写：

- 字段别名复杂。
- 一个对象里嵌套另一个对象。
- 一对多集合。
- 多表 join。
- 需要指定 typeHandler。

## association：一对一映射

比如订单里有用户信息。

```java
public class Order {
    private Long id;
    private String orderNo;
    private User user;
}
```

XML：

```xml
<resultMap id="OrderResultMap" type="com.example.domain.Order">
  <id property="id" column="order_id"/>
  <result property="orderNo" column="order_no"/>

  <association property="user" javaType="com.example.domain.User">
    <id property="id" column="user_id"/>
    <result property="userName" column="user_name"/>
    <result property="email" column="email"/>
  </association>
</resultMap>

<select id="selectOrderById" resultMap="OrderResultMap">
  select
    o.id as order_id,
    o.order_no,
    u.id as user_id,
    u.user_name,
    u.email
  from orders o
  left join user u on o.user_id = u.id
  where o.id = #{id}
</select>
```

`association` 用来映射对象里的另一个对象。

## collection：一对多映射

比如一个用户有多个订单。

```java
public class User {
    private Long id;
    private String userName;
    private List<Order> orders;
}
```

XML：

```xml
<resultMap id="UserWithOrdersResultMap" type="com.example.domain.User">
  <id property="id" column="user_id"/>
  <result property="userName" column="user_name"/>

  <collection property="orders" ofType="com.example.domain.Order">
    <id property="id" column="order_id"/>
    <result property="orderNo" column="order_no"/>
  </collection>
</resultMap>

<select id="selectUserWithOrders" resultMap="UserWithOrdersResultMap">
  select
    u.id as user_id,
    u.user_name,
    o.id as order_id,
    o.order_no
  from user u
  left join orders o on u.id = o.user_id
  where u.id = #{id}
</select>
```

这里一定要注意列别名。

如果用户表和订单表都有 `id`，不加别名就很容易映射乱。

## resultMap 可以继承

如果多个 resultMap 有重复字段，可以用 `extends`。

```xml
<resultMap id="UserBaseMap" type="com.example.domain.User">
  <id property="id" column="id"/>
  <result property="userName" column="user_name"/>
</resultMap>

<resultMap id="UserDetailMap" type="com.example.domain.User" extends="UserBaseMap">
  <result property="email" column="email"/>
  <result property="phone" column="phone"/>
</resultMap>
```

不要滥用继承。字段确实稳定复用时再用，否则看 XML 会很绕。

## 常见坑

### 1. resultType 和 resultMap 同时写

不要这样写：

```xml
<select id="selectById" resultType="User" resultMap="UserResultMap">
```

二选一。

### 2. 多表 join 不起别名

不建议：

```sql
select u.id, o.id from user u left join orders o on ...
```

建议：

```sql
select u.id as user_id, o.id as order_id from user u left join orders o on ...
```

### 3. collection 缺少 id 映射

一对多映射里，父对象和子对象的主键都应该用 `<id>` 标出来。

否则 MyBatis 组装对象时可能出现重复或合并异常。

## 小结

`resultType` 适合简单结果。

`resultMap` 适合复杂映射。

可以先记住四个标签：

```text
id          主键字段
result      普通字段
association 一对一对象
collection  一对多集合
```
