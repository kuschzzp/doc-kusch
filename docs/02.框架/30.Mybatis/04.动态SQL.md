# MyBatis 动态 SQL

官方文档：[MyBatis 3 - Dynamic SQL](https://mybatis.org/mybatis-3/dynamic-sql.html)

动态 SQL 是 MyBatis 很常用的一块。

它解决的问题很简单：SQL 条件不是固定的。

比如搜索用户时，用户名、手机号、状态、时间范围都可能为空。如果手动拼字符串，很容易漏空格、漏 `AND`、多逗号。MyBatis 用 XML 标签把这些情况处理掉。

官方文档里提到的核心标签主要是：

```text
if
choose / when / otherwise
trim / where / set
foreach
```

## if：按条件拼接

```xml
<select id="selectUsers" resultType="com.example.domain.User">
  select id, user_name, email, status
  from user
  where 1 = 1
  <if test="userName != null and userName != ''">
    and user_name like concat('%', #{userName}, '%')
  </if>
  <if test="status != null">
    and status = #{status}
  </if>
</select>
```

`test` 里使用的是 OGNL 表达式。

但这种 `where 1 = 1` 写法虽然常见，不是最优雅。MyBatis 提供了 `<where>`。

## where：自动处理 WHERE 和 AND

```xml
<select id="selectUsers" resultType="com.example.domain.User">
  select id, user_name, email, status
  from user
  <where>
    <if test="userName != null and userName != ''">
      and user_name like concat('%', #{userName}, '%')
    </if>
    <if test="status != null">
      and status = #{status}
    </if>
  </where>
</select>
```

`where` 做了两件事：

- 里面有内容时，才加 `WHERE`。
- 如果开头是 `AND` 或 `OR`，会自动去掉。

这比手写 `where 1 = 1` 干净。

## choose：多个条件只选一个

`choose` 类似 Java 里的 `switch`。

```xml
<select id="selectUsers" resultType="com.example.domain.User">
  select id, user_name, email, status
  from user
  <where>
    <choose>
      <when test="userName != null and userName != ''">
        user_name like concat('%', #{userName}, '%')
      </when>
      <when test="email != null and email != ''">
        email = #{email}
      </when>
      <otherwise>
        status = 1
      </otherwise>
    </choose>
  </where>
</select>
```

这个 SQL 的意思是：

- 有用户名就按用户名查。
- 没有用户名但有邮箱，就按邮箱查。
- 都没有，就查启用状态的用户。

## set：动态更新字段

更新时最容易出现的问题是多一个逗号。

```xml
<update id="updateUser">
  update user
  <set>
    <if test="userName != null">user_name = #{userName},</if>
    <if test="email != null">email = #{email},</if>
    <if test="status != null">status = #{status},</if>
  </set>
  where id = #{id}
</update>
```

`set` 会自动加 `SET`，并去掉最后多余的逗号。

## trim：更通用的 where / set

`where` 和 `set` 本质上都是特殊的 `trim`。

等价 `where`：

```xml
<trim prefix="WHERE" prefixOverrides="AND |OR ">
  ...
</trim>
```

等价 `set`：

```xml
<trim prefix="SET" suffixOverrides=",">
  ...
</trim>
```

一般用 `where`、`set` 就够了。只有遇到特殊 SQL 片段时再用 `trim`。

## foreach：处理 IN 查询

```xml
<select id="selectByIds" resultType="com.example.domain.User">
  select id, user_name, email
  from user
  where id in
  <foreach collection="ids" item="id" open="(" separator="," close=")">
    #{id}
  </foreach>
</select>
```

接口：

```java
List<User> selectByIds(@Param("ids") List<Long> ids);
```

`foreach` 常用属性：

| 属性 | 说明 |
| --- | --- |
| `collection` | 要遍历的集合 |
| `item` | 当前元素变量名 |
| `index` | 当前下标或 key |
| `open` | 开始拼接内容 |
| `separator` | 分隔符 |
| `close` | 结束拼接内容 |

MyBatis 3.5.9 以后还有一个 `nullable` 相关配置，可以控制 foreach 遇到 null 时的行为。

## bind：提前绑定变量

如果你不想在 Java 里拼 `%keyword%`，可以用 `bind`。

```xml
<select id="selectByName" resultType="com.example.domain.User">
  <bind name="pattern" value="'%' + userName + '%'"/>
  select id, user_name, email
  from user
  where user_name like #{pattern}
</select>
```

这样比使用 `${}` 拼接安全。

## 注解里也能写动态 SQL

如果用注解，可以用 `<script>`。

```java
@Update({
    "<script>",
    "update user",
    "<set>",
    "  <if test='userName != null'>user_name = #{userName},</if>",
    "  <if test='email != null'>email = #{email},</if>",
    "</set>",
    "where id = #{id}",
    "</script>"
})
int updateUser(User user);
```

不过复杂 SQL 我还是建议放 XML。注解里写长 SQL，可读性会很差。

## 小结

动态 SQL 先记住这几个：

```text
if       单个条件
choose   多个条件选一个
where    自动处理 WHERE / AND
set      自动处理 SET / 逗号
foreach  遍历集合，常用于 IN
bind     绑定临时变量
```

写动态 SQL 时，尽量少用 `${}`，优先用 `#{}`。
