# MyBatis 缓存机制

官方文档：[MyBatis 3 - Mapper XML Files / cache](https://mybatis.org/mybatis-3/sqlmap-xml.html)

MyBatis 有两类缓存容易混：

```text
一级缓存：SqlSession 级别，默认存在
二级缓存：Mapper namespace 级别，需要配置
```

## 一级缓存

一级缓存是 SqlSession 级别的本地缓存。

同一个 SqlSession 里，执行相同查询，MyBatis 可以复用前一次结果，避免重复查数据库。

官方配置里有一个相关项：

```xml
<setting name="localCacheScope" value="SESSION"/>
```

默认是 `SESSION`。

如果改成 `STATEMENT`：

```xml
<setting name="localCacheScope" value="STATEMENT"/>
```

那缓存只在一次 statement 执行期间有效，不会跨多次查询共享。

一般项目里很少专门改它，知道它存在就行。

## 二级缓存

二级缓存是 mapper namespace 级别的缓存。

开启方式很简单，在 Mapper XML 里加：

```xml
<mapper namespace="com.example.mapper.UserMapper">
  <cache/>

  <select id="selectById" resultType="com.example.domain.User">
    select id, user_name, email
    from user
    where id = #{id}
  </select>
</mapper>
```

官方文档说，加上 `<cache/>` 后默认效果包括：

- 当前 mapper 里的 select 结果会被缓存。
- 当前 mapper 里的 insert、update、delete 会清空缓存。
- 默认使用 LRU 淘汰策略。
- 默认缓存 1024 个引用。
- 默认是 read/write 缓存。

## cache 常用属性

```xml
<cache
  eviction="LRU"
  flushInterval="60000"
  size="512"
  readOnly="true"/>
```

| 属性 | 说明 |
| --- | --- |
| `eviction` | 淘汰策略，比如 `LRU`、`FIFO`、`SOFT`、`WEAK` |
| `flushInterval` | 定时刷新间隔，单位毫秒 |
| `size` | 最多缓存多少对象引用 |
| `readOnly` | 是否只读 |

`readOnly="true"` 性能更好，但返回的是同一个对象引用，调用方不要改它。

如果对象可能被修改，别轻易开只读。

## select 上的 useCache

查询语句可以控制是否使用二级缓存。

```xml
<select id="selectById" resultType="com.example.domain.User" useCache="true">
  select id, user_name, email
  from user
  where id = #{id}
</select>
```

`select` 默认 `useCache="true"`。

如果某个查询不适合缓存，可以关掉：

```xml
<select id="selectLatest" resultType="com.example.domain.User" useCache="false">
  select id, user_name, email
  from user
  order by create_time desc
  limit 10
</select>
```

## flushCache

`insert`、`update`、`delete` 默认会刷新缓存。

`select` 默认不会刷新缓存。

可以手动控制：

```xml
<select id="selectById" resultType="com.example.domain.User" flushCache="false">
  select id, user_name, email
  from user
  where id = #{id}
</select>
```

```xml
<update id="updateUser" flushCache="true">
  update user set email = #{email} where id = #{id}
</update>
```

大多数时候保持默认就行。

## cache-ref：共享缓存

如果多个 mapper 想共享同一个 namespace 的缓存，可以用 `cache-ref`。

```xml
<cache-ref namespace="com.example.mapper.UserMapper"/>
```

这表示当前 mapper 使用 `UserMapper` 的缓存配置。

## 实际项目里要不要开二级缓存？

我的习惯是：谨慎。

原因是二级缓存很容易和业务实时性冲突。

比如用户信息、权限、库存、订单状态，这些数据变化频繁，缓存不一致会很麻烦。

更常见的做法是：

- MyBatis 一级缓存保持默认。
- 二级缓存默认不开。
- 真要缓存，优先在业务层用 Redis、Caffeine，并明确缓存 key、过期时间、失效策略。

当然，如果是字典表、配置表、变化很少的数据，MyBatis 二级缓存也可以考虑。

## 小结

可以这样记：

```text
一级缓存：SqlSession 级别，默认有
二级缓存：Mapper namespace 级别，需要 <cache/>
select 默认 useCache=true
写操作默认 flushCache=true
```

缓存不是越多越好。先保证数据正确，再考虑性能。
