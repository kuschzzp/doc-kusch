# Mapper XML 基本写法

官方文档：[MyBatis 3 - Mapper XML Files](https://mybatis.org/mybatis-3/sqlmap-xml.html)

MyBatis 真正干活的地方，大部分都在 Mapper XML。

配置文件告诉 MyBatis “去哪里找 mapper”，Mapper XML 才是写 SQL、映射结果的地方。

## 一个最小 Mapper

接口：

```java
package com.example.mapper;

import com.example.domain.User;

public interface UserMapper {
    User selectById(Long id);
}
```

XML：

```xml
<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper
  PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
  "https://mybatis.org/dtd/mybatis-3-mapper.dtd">

<mapper namespace="com.example.mapper.UserMapper">
  <select id="selectById" parameterType="long" resultType="com.example.domain.User">
    select id, username, email
    from user
    where id = #{id}
  </select>
</mapper>
```

这里最重要的两个点：

- `namespace` 要对应 mapper 接口全限定名。
- `select` 的 `id` 要对应接口方法名。

也就是说：

```text
com.example.mapper.UserMapper.selectById
```

就是 MyBatis 最终定位 SQL 的 statement id。

## select

常见写法：

```xml
<select id="selectById" parameterType="long" resultType="com.example.domain.User">
  select id, username, email
  from user
  where id = #{id}
</select>
```

`select` 常用属性：

| 属性 | 说明 |
| --- | --- |
| `id` | statement id，一般和接口方法名一致 |
| `parameterType` | 参数类型，很多时候可以省略 |
| `resultType` | 返回类型，简单映射可以用它 |
| `resultMap` | 复杂映射用它，不能和 `resultType` 同时使用 |
| `flushCache` | 是否清空缓存，select 默认 `false` |
| `useCache` | 是否使用二级缓存，select 默认 `true` |
| `timeout` | SQL 超时时间，单位秒 |
| `fetchSize` | 给 JDBC driver 的抓取行数建议 |

简单对象可以用 `resultType`。

一旦涉及字段名不一致、关联对象、集合对象，就应该换成 `resultMap`。

## insert

```xml
<insert id="insertUser" parameterType="com.example.domain.User" useGeneratedKeys="true" keyProperty="id">
  insert into user(username, email)
  values(#{username}, #{email})
</insert>
```

`useGeneratedKeys="true"` 表示使用 JDBC 的自动生成主键能力。

`keyProperty="id"` 表示把生成的主键回填到 Java 对象的 `id` 属性上。

如果数据库不支持这种方式，也可以用 `selectKey`。

```xml
<insert id="insertUser" parameterType="com.example.domain.User">
  <selectKey keyProperty="id" resultType="long" order="BEFORE">
    select nextval('user_id_seq')
  </selectKey>
  insert into user(id, username, email)
  values(#{id}, #{username}, #{email})
</insert>
```

## update

```xml
<update id="updateEmail">
  update user
  set email = #{email}
  where id = #{id}
</update>
```

接口可以这样写：

```java
int updateEmail(@Param("id") Long id, @Param("email") String email);
```

返回值一般用 `int`，表示影响行数。

## delete

```xml
<delete id="deleteById">
  delete from user
  where id = #{id}
</delete>
```

接口：

```java
int deleteById(Long id);
```

## sql 和 include

如果很多 SQL 都要查同一批字段，不要到处复制。

```xml
<sql id="Base_Column_List">
  id, username, email, create_time
</sql>

<select id="selectById" resultType="com.example.domain.User">
  select <include refid="Base_Column_List"/>
  from user
  where id = #{id}
</select>
```

`sql` 只是可复用 SQL 片段，不是可执行语句。

## #{ } 和 ${ } 的区别

这个很重要。

`#{}` 是预编译参数，占位符方式，能防 SQL 注入。

```xml
where username = #{username}
```

最终类似：

```sql
where username = ?
```

`${}` 是字符串替换。

```xml
order by ${columnName}
```

它会直接拼进 SQL，有注入风险。

所以原则很简单：

```text
值用 #{}
表名、字段名这类 SQL 结构才可能用 ${}，而且必须白名单校验
```

## resultType 和 resultMap 怎么选？

简单映射：

```xml
<select id="selectById" resultType="com.example.domain.User">
  select id, username, email from user where id = #{id}
</select>
```

复杂映射：

```xml
<select id="selectById" resultMap="UserResultMap">
  select id, user_name, email from user where id = #{id}
</select>
```

官方文档也强调：`resultType` 和 `resultMap` 二选一，不要同时写。

## 小结

Mapper XML 先掌握这几个标签就够用：

```text
select / insert / update / delete
resultMap
sql / include
cache
```

SQL 本身还是 SQL，MyBatis 只是帮你把参数绑定、结果映射、接口调用这些重复代码省掉。
