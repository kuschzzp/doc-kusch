# SqlSession 与 Mapper 代理

官方文档：[MyBatis 3 - Java API](https://mybatis.org/mybatis-3/java-api.html)

如果只在 Spring Boot 里用 MyBatis，平时很少直接写 `SqlSession`。

但理解它很有必要，因为 MyBatis 的核心调用链就是围绕它来的。

## 三个核心对象

```text
SqlSessionFactoryBuilder -> SqlSessionFactory -> SqlSession
```

### SqlSessionFactoryBuilder

负责创建 `SqlSessionFactory`。

通常只在启动时用一次。

```java
String resource = "mybatis-config.xml";
InputStream inputStream = Resources.getResourceAsStream(resource);
SqlSessionFactory sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);
```

### SqlSessionFactory

负责创建 `SqlSession`。

它应该是应用级别的对象，通常整个应用只需要一个。

```java
SqlSession session = sqlSessionFactory.openSession();
```

### SqlSession

真正执行 SQL、获取 mapper、管理事务的是 `SqlSession`。

官方文档也说，MyBatis 的主要 Java 接口就是 `SqlSession`。

## 直接使用 SqlSession

```java
try (SqlSession session = sqlSessionFactory.openSession()) {
    User user = session.selectOne(
        "com.example.mapper.UserMapper.selectById",
        1L
    );
    System.out.println(user);
}
```

这种写法可以用，但字符串太硬。

```text
com.example.mapper.UserMapper.selectById
```

一旦写错，编译期发现不了。

所以实际项目更常用 Mapper 代理。

## Mapper 代理

接口：

```java
package com.example.mapper;

import com.example.domain.User;

public interface UserMapper {
    User selectById(Long id);
}
```

XML：

```xml
<mapper namespace="com.example.mapper.UserMapper">
  <select id="selectById" resultType="com.example.domain.User">
    select id, user_name, email
    from user
    where id = #{id}
  </select>
</mapper>
```

调用：

```java
try (SqlSession session = sqlSessionFactory.openSession()) {
    UserMapper mapper = session.getMapper(UserMapper.class);
    User user = mapper.selectById(1L);
    System.out.println(user);
}
```

MyBatis 会为 `UserMapper` 生成代理对象。你调用接口方法，MyBatis 根据：

```text
接口全限定名 + 方法名
```

找到对应 XML statement。

也就是：

```text
com.example.mapper.UserMapper.selectById
```

## 事务提交

默认 `openSession()` 不是自动提交。

```java
try (SqlSession session = sqlSessionFactory.openSession()) {
    UserMapper mapper = session.getMapper(UserMapper.class);
    mapper.insertUser(user);
    session.commit();
}
```

如果忘了 `commit()`，写操作可能不会真正提交。

也可以开启自动提交：

```java
try (SqlSession session = sqlSessionFactory.openSession(true)) {
    UserMapper mapper = session.getMapper(UserMapper.class);
    mapper.insertUser(user);
}
```

但在业务项目里，我更建议交给 Spring 事务管理，而不是手动控制。

## ExecutorType

`openSession` 可以指定执行器类型。

官方文档里提到几个常见类型：

| 类型 | 含义 |
| --- | --- |
| `SIMPLE` | 每次执行都创建新的 PreparedStatement |
| `REUSE` | 复用 PreparedStatement |
| `BATCH` | 批处理更新 |

批量插入时可能会用：

```java
try (SqlSession session = sqlSessionFactory.openSession(ExecutorType.BATCH)) {
    UserMapper mapper = session.getMapper(UserMapper.class);
    for (User user : users) {
        mapper.insertUser(user);
    }
    session.commit();
}
```

不过批处理也要注意内存和事务大小。数据量特别大时，不要一口气全塞进去，可以分批提交。

## Spring 项目里发生了什么？

Spring Boot + MyBatis 里，我们通常只写：

```java
@Mapper
public interface UserMapper {
    User selectById(Long id);
}
```

然后直接注入：

```java
@Service
public class UserService {
    private final UserMapper userMapper;

    public UserService(UserMapper userMapper) {
        this.userMapper = userMapper;
    }

    public User getUser(Long id) {
        return userMapper.selectById(id);
    }
}
```

底层还是：

```text
SqlSession -> getMapper -> Mapper 代理 -> 执行 mapped statement
```

只是 SqlSession 的创建、关闭、事务绑定都被 Spring 接管了。

## 小结

MyBatis 的 Java API 可以这样记：

```text
SqlSessionFactoryBuilder  负责构建工厂
SqlSessionFactory         负责创建会话
SqlSession                负责执行 SQL 和管理事务
Mapper 代理               负责把接口方法映射到 SQL statement
```

日常写业务时，你可能只接触 Mapper 接口。

但排查事务、缓存、批处理、连接问题时，最终还是要回到 SqlSession 这层理解。
