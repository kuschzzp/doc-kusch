## 一句话总览

> Agent Framework（框架）= 怎么写 Agent  
> Agent Runtime（运行时）= 怎么稳定地跑 Agent  
> Agent Harness（工具箱 / 成品）= 直接拿来用的 Agent  

LangChain 团队试图用这三个词，解释它们自家项目（以及整个 Agent 生态）之间的层级关系。

---

## 一、Agent Framework（代理框架）——「怎么写 Agent」

代表：LangChain

### 核心定位

> 提供 抽象（Abstractions），帮助你更容易构建 Agent。

### 它解决的问题

* 怎么组织 Prompt？  
* Agent 的「思考 → 行动 → 观察 → 再思考」循环怎么写？  
* Tool / Memory / Output 怎么统一？  
* 新人怎么快速上手？  

### 本质理解

👉 Agent Framework 是一套“编程范式 + 抽象模型”

它像是：

* Web 里的 Spring / Rails  
* 前端里的 React  

好处：

* 快速上手  
* 统一开发方式  
* 降低心智负担  

缺点：

* 抽象不好会“挡住你”  
* 高级玩法可能不够灵活  

### 作者提到的其他 Framework

* Vercel AI SDK
* CrewAI
* OpenAI Agents SDK
* Google ADK
* LlamaIndex

---

## 二、Agent Runtime（代理运行时）——「怎么把 Agent 跑稳」

代表：LangGraph

### 核心定位

> 面向 生产环境（Production） 的 Agent 执行引擎

### 它解决的问题

当 Agent 真正上线后，会遇到：

* ❌ 进程挂了怎么办？
* ❌ 执行到一半中断怎么办？
* ❌ 如何支持流式输出？
* ❌ 如何做 Human-in-the-loop（人工介入）？
* ❌ 如何保存/恢复上下文（线程级、跨线程）？

### 本质理解

👉 Agent Runtime = Agent 世界的“操作系统 / 工作流引擎”

它更像：

* Temporal
* Inngest
* Durable Execution Engine

### 关键特点

* Durable Execution（可持久执行）
* 状态可恢复
* 更低层、更偏基础设施
* 可以支撑多个 Framework

👉 LangChain 1.0 本身就是跑在 LangGraph 之上的

---

## 三、Agent Harness（代理工具箱 / 成品）——「直接拿来用」

代表：DeepAgents

### 核心定位

> 比 Framework 还高一层，自带“电池”的 Agent

### 它都帮你做了什么？

* 默认 Prompt（不用从 0 写）
* 约定好的 Tool 调用方式
* 规划（Planning）能力
* 文件系统访问
* 更“像人”的 Agent 行为

### 本质理解

👉 Agent Harness = 已经组装好的“通用 Agent”

它更像：

* 一个「可直接使用的智能体」
* 而不是一个“搭积木的工具”

### 类比

作者给了一个非常形象的比喻：

> DeepAgents ≈ 通用版的 Claude Code

你可以理解为：

* Claude Code：偏代码
* DeepAgents：偏通用（代码 + 文件 + 规划 + 多工具）

### 延伸理解

其实很多 Coding CLI 工具：

* Cursor
* Claude Code
* Copilot CLI

某种意义上，都算 Agent Harness

---

## 四、三者关系一图看懂（心智模型）

```
┌─────────────────────────┐
│   Agent Harness         │  ← 直接可用（DeepAgents）
│  （成品 / 工具箱）       │
├─────────────────────────┤
│   Agent Framework       │  ← 开发范式（LangChain）
│  （抽象 / API）          │
├─────────────────────────┤
│   Agent Runtime         │  ← 执行引擎（LangGraph）
│  （稳定 / 持久 / 状态）  │
└─────────────────────────┘
```

或者一句话：

> Harness 用 Framework 写，Framework 跑在 Runtime 上

---

## 五、什么时候用哪个？

### ✅ 用 Agent Framework（LangChain）

* 你要自己设计 Agent 逻辑
* 你在做 业务型 Agent
* 你需要灵活组合 Prompt / Tool / Memory

### ✅ 用 Agent Runtime（LangGraph）

* 你要 Agent 上生产
* 你需要：

    * 状态恢复
    * 长任务
    * 人工介入
    * 流式执行

### ✅ 用 Agent Harness（DeepAgents）

* 你想 直接用一个“强力 Agent”
* 你不想从零设计 Prompt 和行为
* 类似“直接用 Claude Code 干活”

## 六、作者的态度（很重要）

作者非常诚实地说了三点：

1. 边界是模糊的

    * LangGraph 既像 Runtime，也像 Framework
2. 这些词还没行业标准
3. 这是一个早期领域

    * 大家还在“发明语言 + 心智模型”

👉 本文的真正价值，不是定义本身  
👉 而是帮你区分「抽象层级」
