# LangGraph 入门

> 官方文档：https://docs.langchain.com/oss/python/langgraph/overview

当 Agent 流程变复杂时，只靠 `create_agent` 不一定够。

LangGraph 适合你需要明确控制：

- 状态
- 分支
- 循环
- 重试
- 人类审批
- 多节点工作流

## 安装

```bash
pip install -U langgraph langchain-openai
```

## 最小图示例

```python
from typing import TypedDict
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    text: str
    result: str


def add_prefix(state: State) -> State:
    return {
        "text": state["text"],
        "result": "处理结果：" + state["text"],
    }

builder = StateGraph(State)
builder.add_node("add_prefix", add_prefix)
builder.add_edge(START, "add_prefix")
builder.add_edge("add_prefix", END)

graph = builder.compile()

result = graph.invoke({"text": "hello", "result": ""})
print(result["result"])
```

这个例子没有调用模型，但展示了 LangGraph 的核心：

```text
State -> Node -> Edge -> New State
```

## 什么时候从 LangChain 升级到 LangGraph？

| 情况 | 是否需要 LangGraph |
| --- | --- |
| 简单 prompt 链 | 不需要 |
| 简单工具 Agent | 不一定需要 |
| 需要条件分支 | 需要 |
| 需要循环直到满足条件 | 需要 |
| 需要中断让人审批 | 需要 |
| 需要持久化恢复 | 需要 |

## 总结

LangChain 更适合快速构建。

LangGraph 更适合你自己掌控执行流程。
