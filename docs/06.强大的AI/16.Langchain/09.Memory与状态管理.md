# Memory 与状态管理

> 官方文档：https://docs.langchain.com/oss/python/langchain/short-term-memory

LangChain 早期有很多 Memory 类。现在更推荐从“状态管理”的角度理解 Memory。

## Agent 短期记忆

`create_agent` 使用 checkpointer 保存会话状态。

```python
from langchain.agents import create_agent
from langgraph.checkpoint.memory import MemorySaver

agent = create_agent(
    model="openai:gpt-4o-mini",
    tools=[],
    system_prompt="你是一个中文助手。",
    checkpointer=MemorySaver(),
)

config = {"configurable": {"thread_id": "user-001"}}

agent.invoke(
    {"messages": [{"role": "user", "content": "我叫李雷。"}]},
    config=config,
)

result = agent.invoke(
    {"messages": [{"role": "user", "content": "我叫什么？"}]},
    config=config,
)

print(result["messages"][-1].content)
```

同一个 `thread_id` 会共享历史，不同 `thread_id` 互相隔离。

## RunnableWithMessageHistory

普通聊天链可以用 `RunnableWithMessageHistory`。

```python
from langchain_core.chat_history import InMemoryChatMessageHistory
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_core.runnables.history import RunnableWithMessageHistory
from langchain_openai import ChatOpenAI

prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个中文助手。"),
    MessagesPlaceholder("history"),
    ("human", "{input}"),
])

model = ChatOpenAI(model="gpt-4o-mini", temperature=0)
chain = prompt | model

store: dict[str, InMemoryChatMessageHistory] = {}

def get_history(session_id: str) -> InMemoryChatMessageHistory:
    if session_id not in store:
        store[session_id] = InMemoryChatMessageHistory()
    return store[session_id]

chain_with_history = RunnableWithMessageHistory(
    chain,
    get_history,
    input_messages_key="input",
    history_messages_key="history",
)

config = {"configurable": {"session_id": "user-001"}}

chain_with_history.invoke({"input": "我叫李雷。"}, config=config)
response = chain_with_history.invoke({"input": "我叫什么？"}, config=config)

print(response.content)
```

## 怎么选？

| 场景 | 推荐 |
| --- | --- |
| Agent 多轮对话 | `checkpointer` + `thread_id` |
| 普通 Runnable 聊天 | `RunnableWithMessageHistory` |
| 复杂状态流转 | LangGraph |
| 长期记忆 | 数据库 / Store / 自定义记忆系统 |

## 总结

Memory 的本质是：

```text
会话 ID -> 找到历史状态 -> 注入上下文 -> 保存新状态
```
