# RAG 基础流程

> 官方文档：https://docs.langchain.com/oss/python/langchain/retrieval

RAG 是 Retrieval Augmented Generation，检索增强生成。

基础流程：

```text
Load -> Split -> Embed -> Store -> Retrieve -> Generate
```

## 最小可运行示例

安装：

```bash
pip install -U langchain langchain-openai
export OPENAI_API_KEY="你的 key"
```

代码：

```python
from langchain_core.documents import Document
from langchain_core.vectorstores import InMemoryVectorStore
from langchain_openai import ChatOpenAI, OpenAIEmbeddings

# 1. 准备文档
docs = [
    Document(page_content="LangChain 是一个构建 LLM 应用的框架。"),
    Document(page_content="RAG 的核心是先检索相关上下文，再让模型基于上下文回答。"),
    Document(page_content="Retriever 负责根据问题取回相关文档。"),
]

# 2. 向量化并存储
embeddings = OpenAIEmbeddings(model="text-embedding-3-small")
vectorstore = InMemoryVectorStore.from_documents(docs, embeddings)

# 3. 创建检索器
retriever = vectorstore.as_retriever(search_kwargs={"k": 2})

# 4. 检索
query = "RAG 是什么？"
retrieved_docs = retriever.invoke(query)
context = "\n\n".join(doc.page_content for doc in retrieved_docs)

# 5. 生成
model = ChatOpenAI(model="gpt-4o-mini", temperature=0)
response = model.invoke([
    {"role": "system", "content": f"请只基于以下上下文回答：\n\n{context}"},
    {"role": "user", "content": query},
])

print(response.content)
```

## 不要一开始就用旧封装

旧教程常见：

```python
# RetrievalQA.from_chain_type(...)
```

新项目更建议显式写出 RAG 流程。这样更容易加入 metadata filter、rerank、query rewrite、citation 和 LangSmith trace。

## 总结

RAG 的关键不是“把文档塞给模型”，而是：

> 先用 Retriever 找到相关上下文，再让模型基于上下文生成答案。
