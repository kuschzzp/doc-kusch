# Tools 工具定义

Tool 是模型可以调用的外部能力。

比如：

- 查天气
- 查询数据库
- 调用 HTTP API
- 搜索文档
- 创建工单

## 定义工具

```python
from langchain_core.tools import tool

@tool
def get_weather(city: str) -> str:
    """查询指定城市的天气。

    Args:
        city: 城市名称，例如 北京、上海、杭州。
    """
    return f"{city} 今天晴天，气温 30°C。"

print(get_weather.name)
print(get_weather.description)
print(get_weather.args)
```

## 工具描述很重要

模型会根据工具描述决定是否调用工具。

不好的写法：

```python
from langchain_core.tools import tool

@tool
def bad_tool(input: str) -> str:
    """处理输入。"""
    return input
```

好的写法：

```python
from langchain_core.tools import tool

@tool
def search_order(order_id: str) -> str:
    """查询订单状态。

    当用户询问订单是否发货、物流状态、订单进度时使用。

    Args:
        order_id: 订单编号，例如 OD202605070001。
    """
    return f"订单 {order_id} 已发货。"
```

## 在 Agent 中使用工具

```python
from langchain.agents import create_agent
from langchain_core.tools import tool

@tool
def get_weather(city: str) -> str:
    """查询指定城市的天气。

    Args:
        city: 城市名称。
    """
    return f"{city} 今天晴天，气温 30°C。"

agent = create_agent(
    model="openai:gpt-4o-mini",
    tools=[get_weather],
    system_prompt="你是一个中文助手。需要天气信息时调用工具。",
)

result = agent.invoke({
    "messages": [{"role": "user", "content": "北京今天的天气怎么样？"}]
})

print(result["messages"][-1].content)
```

## 总结

Tool 的关键不是 Python 函数本身，而是：

- 名称清晰
- 描述明确
- 参数可理解
- 返回值稳定
