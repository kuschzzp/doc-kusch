# LangChain 生态总览

> 官方文档：https://docs.langchain.com/oss/python/langchain/overview

LangChain 现在不应该再理解成一个“大而全框架”，而应该理解成一套分层生态。

```text
LangChain   基础组件：模型、Prompt、Tools、RAG、简单 Agent
LangGraph   编排层：状态、分支、循环、持久化、人类审批
Deep Agents 高层 Agent：计划、文件、子 Agent、Skills、长期任务
LangSmith   可观测、调试、数据集、评估、部署
LangServe   将 Runnable 暴露为 HTTP API 的轻量部署工具
```

## 什么时候用哪一层？

| 需求 | 推荐 |
| --- | --- |
| 调模型、写 Prompt、组合 Chain | LangChain |
| 简单工具调用 Agent | LangChain `create_agent` |
| 复杂流程、状态机、循环、审批 | LangGraph |
| 长任务、文件操作、子 Agent、Skills | Deep Agents |
| 调试、Tracing、评估 | LangSmith |
| 快速暴露 HTTP API | LangServe |

## 最小可运行示例

安装：

```bash
pip install -U langchain langchain-openai
export OPENAI_API_KEY="你的 key"
```

代码：

```python
from langchain_openai import ChatOpenAI

model = ChatOpenAI(model="gpt-4o-mini", temperature=0)
response = model.invoke("用一句话解释 LangChain 是什么。")
print(response.content)
```

## 当前学习路线

建议顺序：

```text
ChatModel -> Prompt -> Runnable -> OutputParser -> Tools -> Agent -> Memory -> RAG -> LangGraph -> LangSmith -> 部署
```

不要从旧的 `initialize_agent`、`AgentType`、`RetrievalQA` 开始学习。它们在旧教程里很常见，但新项目应优先使用 LangChain 1.x 的推荐写法。
