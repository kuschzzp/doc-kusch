# Agent 与 create_agent

> 官方文档：https://docs.langchain.com/oss/python/langchain/agents

当前 LangChain 推荐使用 `create_agent` 创建 Agent。

不要再从旧的 `initialize_agent`、`AgentType` 开始。

## 最小 Agent

```python
from langchain.agents import create_agent

agent = create_agent(
    model="openai:gpt-4o-mini",
    tools=[],
    system_prompt="你是一个简洁的中文助手。",
)

result = agent.invoke({
    "messages": [{"role": "user", "content": "用一句话解释 Agent 是什么。"}]
})

print(result["messages"][-1].content)
```

## 带工具的 Agent

```python
from langchain.agents import create_agent
from langchain_core.tools import tool

@tool
def add(a: float, b: float) -> float:
    """计算两个数字之和。

    Args:
        a: 第一个数字。
        b: 第二个数字。
    """
    return a + b

agent = create_agent(
    model="openai:gpt-4o-mini",
    tools=[add],
    system_prompt="你是一个数学助手。需要计算时使用工具。",
)

result = agent.invoke({
    "messages": [{"role": "user", "content": "12.5 加 7.3 等于多少？"}]
})

print(result["messages"][-1].content)
```

## 返回结果怎么读？

`create_agent` 返回的是完整状态，不是单条消息。

正确：

```python
print(result["messages"][-1].content)
```

错误：

```python
# print(result.content)
```

## 什么时候用 Agent？

适合：

- 模型需要自己决定是否调用工具
- 有多个工具可选
- 步骤不固定
- 需要根据中间结果继续行动

不适合：

- 固定流程
- 不需要工具选择
- 简单 prompt -> model -> parser

固定流程优先用 Runnable 链。

## 总结

当前推荐模式：

```text
@tool 定义工具 -> create_agent 创建 Agent -> messages 输入 -> messages 输出
```
